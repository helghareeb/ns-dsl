# ns-dsl evaluation data

Evaluation outputs for *"A decentralized neutrosophic decision layer for global-state freshness in
microservices architectures"* (Scientific Reports, under review).

All outputs are generated by the evaluation simulator under a **fixed, content-hashed calibration**
and the **recorded random seeds** described in the paper's Methods. The calibration's canonical
SHA-256 is `16dc5ef33908a3723eb896b1bba1129ed147643f8a3bef4a55c459f9ec836070`, embedded in the header
of every result CSV; the single canonical aggregator refuses to mix outputs from a different
calibration.

## Contents

- `results/raw/raw_seed20260615.csv … raw_seed20260619.csv` — raw per-trial outputs, one file per
  master seed (5 seeds). Each row is one trial of one (system, scenario, configuration). Header lines
  record the calibration SHA-256, the code git SHA, the seed, and the schema version.
- `results/tables/*.csv` — the aggregated summaries that underlie every figure and table
  (per-configuration outcomes, Holm-corrected pairwise comparisons, multi-seed replication,
  sensitivity, scalability, adversarial robustness, threshold calibration, weight tuning, measured
  latency/throughput/WAN, clone convergence).
- `results/figures/*` — the generated figures (PDF + PNG).
- `config/calibration.json` — the immutable calibration (its canonical hash matches the CSV headers).

## Regenerating

With the software bundled in this same Zenodo record (`nsdsl-software-v1.zip`):

```bash
pip install -e '.[dev]'
PYTHONPATH=src:. python -m experiments.run_all          # raw per-trial CSVs (from the recorded seeds)
PYTHONPATH=src:. python -m experiments.analyze_results  # -> results/tables/*.csv
PYTHONPATH=src:. python -m experiments.make_figures     # -> results/figures/*
```
